# Logística - Índice de Documentación

Esta carpeta contiene toda la documentación sobre la arquitectura, diseño y reglas de negocio del sistema de logística.

## Documentos Disponibles

- **[Descripción y Alcance](DESCRIPCION_Y_ALCANCE.md):** Visión general del proyecto, contexto y funcionalidades principales.
- **[Arquitectura por Capas](ARQUITECTURA_POR_CAPAS.md):** Explicación del patrón de diseño DDD (Domain-Driven Design) utilizado en el proyecto y separación de responsabilidades.
- **[Modelo de Dominio](MODELO_DE_DOMINIO.md):** Detalles sobre las entidades principales (`Shipment`, `Center`, `Route`) y sus invariantes.
- **[Casos de Uso](CASOS_DE_USO.md):** Catálogo de todas las operaciones soportadas por el sistema (registro, asignación, despacho, etc.).
- **[Contrato Repositorio](CONTRATO_REPOSITORIO.md):** Definición de interfaces para la persistencia de datos y abstracción de la base de datos.
- **[Reglas de Negocio](REGLAS_DE_NEGOCIO.md):** Listado exhaustivo de todas las restricciones lógicas y validaciones implementadas en el dominio.
- **[Datos Iniciales](DATOS_INICIALES.md):** Especificación de los datos pre-cargados al iniciar la aplicación (Seed Data).
- **[Ejecución](EJECUCION.md):** Guía rápida sobre cómo arrancar la aplicación y ejemplos de uso desde la terminal.
- **[Tests y Pasos](TESTS_Y_PASOS.md):** Explicación de las pruebas unitarias y su ejecución.
- **[Troubleshooting](TROUBLESHOOTING.md):** Guía de resolución de problemas comunes.
